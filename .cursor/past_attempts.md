# Past Attempts Log

> Purpose: Track mistakes, failed strategies, and lessons learned.

## Example Entry
### [YYYY-MM-DD] Task: <Task description>
- Problem: <What went wrong>
- Fix: <Corrective action>
- Notes: <Additional guidance>

## Guidelines for Logging
- Each failed execution must append a new entry.
- Include the context of the task, code snippets if needed.
- Always cross-reference with `system_overview.md` to avoid repeated mistakes.
- Optional: categorize by component, file, or workflow stage.
